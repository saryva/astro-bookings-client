# PRD — AstroBookings Client

> 📝 Generated by Agent 1 (Analyst) — fill this running `1-analyst.agent.md` with your AI tool.

## 1. Executive Summary

AstroBookings Client is an Angular Single Page Application that allows customers
to browse available rocket launches and book seats on them. It consumes the
AstroBookings REST API as its sole data source.

**Vision**: Make space travel booking as simple as buying a flight ticket online.

## 2. User Personas

### Persona A — The Space Tourist (Customer)
- **Name**: Alex
- **Goal**: Find an upcoming rocket launch, check price and availability, book seats.
- **Frustration**: Not knowing if a launch is full or if their booking went through.

## 3. Functional Requirements

### FR-01: Display Available Launches
- **Description**: Show a list of all launches available for booking.
- **Priority**: P0 (Must Have)
- **Acceptance Criteria**:
  - Given the launches page loads, When the API responds, Then all launches are displayed in a list/grid.
  - Given a launch, When rendered, Then it shows: rocket name, date, destination, price per seat, available seats.
  - Given 0 available seats, When the launch is rendered, Then it is marked as "Full" and booking is disabled.
  - Given API returns empty array, When rendered, Then an empty state message is shown.
  - Given API call is in progress, When waiting, Then a loading indicator is shown.
  - Given API call fails, When error occurs, Then a user-friendly error message is shown.

### FR-02: View Launch Details
- **Description**: View the full details of a specific launch before booking.
- **Priority**: P0 (Must Have)
- **Acceptance Criteria**:
  - Given a launch in the list, When the user clicks it, Then the launch detail view is shown.
  - Given the detail view, When rendered, Then all launch fields are displayed.
  - Given the detail view, When rendered, Then a "Book Now" button is present (if seats available).

### FR-03: Book Seats on a Launch
- **Description**: Allow a customer to reserve one or more seats on a launch.
- **Priority**: P0 (Must Have)
- **Acceptance Criteria**:
  - Given a launch with available seats, When the user opens booking form, Then they can select number of seats (1 to available).
  - Given the booking form, When user enters seats > available, Then form is invalid and submit is disabled.
  - Given a valid form, When user submits, Then POST /bookings is called with correct payload.
  - Given a successful booking, When API responds 201, Then a success confirmation is shown.
  - Given a failed booking, When API responds with error, Then an error message is shown.

### FR-04: View My Bookings
- **Description**: Allow a customer to view their existing bookings.
- **Priority**: P1 (Should Have)
- **Acceptance Criteria**:
  - Given a customer ID is provided, When the bookings page loads, Then all their bookings are shown.
  - Given a booking, When rendered, Then it shows: launch name, date, seats booked, total price, status.
  - Given no bookings, When rendered, Then an empty state is shown.

### FR-05: Cancel a Booking
- **Description**: Allow a customer to cancel an existing booking.
- **Priority**: P1 (Should Have)
- **Acceptance Criteria**:
  - Given an existing booking, When user clicks "Cancel", Then a confirmation dialog appears.
  - Given confirmation, When user confirms, Then DELETE /bookings/:id is called.
  - Given successful cancellation, When API responds, Then booking is removed from the list.
